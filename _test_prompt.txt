Построй 2D general-схему SoftXAI как компактную научную архитектурную рамку.

Это не веб-архитектура, не бизнес-процесс и не PlotNeuralNet.
Не использовать шаблон user → web → api → service → db.

Схема должна содержать ровно 9 узлов:

1. Вычислительный уровень
2. Логическое ядро SoftXAI
3. Объяснение и коммуникация
4. XAI 2.0
5. Мягкие измерения
6. Встроенная измеримая объяснимость
7. Метрики SoftXAI
8. Архитектуры XAI 2.0
9. Прикладные области

Смысл узлов:

- Вычислительный уровень:
  CNN, Transformer, Graph Model, ANFIS, гибридные и мультимодальные модели, скрытые представления, attention, латентные признаки

- Логическое ядро SoftXAI:
  концепты предметной области, правила ЕСЛИ–ТО, ограничения, неопределенность, нечеткая логика, нейро-символьный вывод

- Объяснение и коммуникация:
  локальные, глобальные, иерархические, адаптивные объяснения, языковая упаковка, диалог с пользователем, LLM как интерфейс

- Метрики SoftXAI:
  faithfulness, stability, semantic relevance, interpretability, decision utility

- Архитектуры XAI 2.0:
  Type-2 ANFIS, FAN, fuzzy attention, explainable CNN, hierarchical explainable systems

- Прикладные области:
  медицинская диагностика, обработка изображений, физиологические сигналы, экологический мониторинг, стратификация риска, поддержка оператора

Обязательная структура:

Центральная вертикальная ось:
Вычислительный уровень
↓
Логическое ядро SoftXAI
↓
Объяснение и коммуникация

Левый блок:
XAI 2.0
Мягкие измерения
↓
Встроенная измеримая объяснимость

Правый блок:
Метрики SoftXAI
↓
Архитектуры XAI 2.0
↓
Прикладные области

Обязательные связи:
- Вычислительный уровень → Логическое ядро SoftXAI
- Логическое ядро SoftXAI → Объяснение и коммуникация
- XAI 2.0 → Встроенная измеримая объяснимость
- Мягкие измерения → Встроенная измеримая объяснимость
- Встроенная измеримая объяснимость → Логическое ядро SoftXAI
- Логическое ядро SoftXAI → Метрики SoftXAI
- Логическое ядро SoftXAI → Архитектуры XAI 2.0
- Архитектуры XAI 2.0 → Прикладные области

Жёсткие ограничения:
- renderer = general
- layout_hint = general
- строго 2D
- строго 9 узлов
- короткие подписи
- без лишних узлов
- без лишних связей
- без пересечений по возможности
- без длинной горизонтальной гирлянды
- без лесенки
- без web/api/service/db
- без flowchart логики

Итог должен выглядеть как компактная научная схема диссертационной концепции SoftXAI.

Верни только валидный JSON без markdown и без пояснений.
